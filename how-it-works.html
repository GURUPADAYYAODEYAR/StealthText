01

Overview

StealthText combines two powerful techniques — LSB Steganography and AES-256-GCM Encryption — to hide secret messages inside ordinary-looking images. Everything runs entirely in your browser; no data is ever sent to a server.

02

Encoding Process

1. Select Image

Upload a PNG or JPEG image. The image's pixel count determines how much data can be hidden.

2. Write Message

Type your secret message. The app displays the maximum character capacity of your chosen image.

3. Encrypt (Optional)

Add a password to encrypt your message with AES-256-GCM before embedding. This adds a layer of cryptographic security.

4. Download

The encoded image is generated as a lossless PNG. Download and share — it looks identical to the original.

Least Significant Bit (LSB) steganography works by replacing the last bit of each color channel (R, G, B) in a pixel. Since this changes the color value by at most 1 out of 255, the difference is invisible to the human eye.

Each pixel stores 3 bits of data (one per channel). A 1000×1000 image can hide approximately 375,000 characters of data.

04

AES-256-GCM Encryption

🔑

PBKDF2 Key Derivation

Your password is transformed into a 256-bit key using 100,000 iterations of PBKDF2-SHA256 with a random salt.

→
🔒

AES-GCM Encryption

Your message is encrypted with the derived key using a random 12-byte IV, providing both confidentiality and integrity.

→
🖼️

Embed in Image

The salt + IV + ciphertext are combined and hidden inside the image's pixel data via LSB steganography.

05

Decoding Process

To decode, simply upload the encoded image (PNG, JPEG, or even a PDF containing the image). StealthText reads the LSB data from each pixel, extracts the header to determine if encryption was used, and then decrypts (if needed) to reveal the original message.

⚠️
Important: JPEG compression and image resizing will destroy the hidden data. Always share encoded images as lossless PNG files.
